# Responsible Use Guide

---

## QA scores are not performance rankings

A score tells you about one interaction. Using scores to rank agents before the team is calibrated and before enough interactions are scored will produce unfair and misleading results.

Use scores for coaching and improvement, not ranking or performance management in isolation.

---

## Separate system failures from agent failures

If an agent followed AI guidance that was wrong, that is a system failure. Coaching the agent for it obscures the real problem and leaves the AI system unfixed.

Always check the responsibility output before any coaching conversation.

---

## Share the rubric with agents

Agents should know how they are being scored. A QA system applied without transparency damages trust.

---

## CSAT below threshold is not automatically the agent's fault

Always run the CSAT analysis before concluding a low score means something went wrong.

---

## Sample size matters

A single low score is noise. A pattern across five or more interactions is signal.

---

## This system does not make decisions

No score from this system should automatically trigger a performance action. Scores inform human judgment. They do not replace it.
